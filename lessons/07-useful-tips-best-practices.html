Step-by-Step Instructions

📁
Before you cloneChoose the folder where you want the project to be downloaded, then use cd to move there in your terminal. That way, Git will place the repo in a location you can easily find again later.
Examples
# macOS cd ~/Desktop # Windows PowerShell cd Desktop # Another common option on either system cd Documents
1

Clone the repository

Terminal
git clone https://github.com/Diptendu123/git-training-demo.git cd git-training-demo
# Your terminal prompt now shows: git-training-demo
✓
State Check — Run git status. You should see "nothing to commit, working tree clean."
2

Create your branch

Use your name so the branch is easy to identify.

Terminal
git checkout -b add-your-name # e.g. add-jane-smith
✓
State Check — Run git branch. An asterisk should appear next to your new branch name, not next to main.
3

Edit index.html

Open index.html in any text editor or IDE. Find this comment:

index.html — find this line
<!-- ADD YOUR CARD BELOW THIS LINE -->

Copy the card template below and paste it after that comment. Then fill in your own details:

Your card — customise all five values
<div class="card"> <div class="avatar" style="background: linear-gradient(135deg, #f093fb, #f5576c);">AB</div> <h3>Your Full Name</h3> <div class="role">Your Job Title</div> <div class="fun-fact">Fun fact: Something interesting about you!</div> </div>
Customise these 5 things
✓
Initials
Use 2 letters in the avatar div, for example JS.
✓
Gradient colours
Replace #f093fb, #f5576c with any two hex colours you like.
Try one of these starter pairs if you want a quick option:
Blue-teal Purple-pink
✓
Name
Enter your full name.
✓
Role
Enter your job title.
✓
Fun fact
Write one short line about yourself.
💡
Preview before you push — open index.html in your browser to see your card exactly as it will appear on the live page.
4

Stage and commit your changes

Terminal
git add index.html git commit -m "feat: add [your name] to team page" # e.g. "feat: add Jane Smith to team page"
✓
State Check — Run git log --oneline. Your commit should appear at the top of the list.
5

Push your branch

Terminal
git push origin add-your-name # match your branch name exactly
6

Open a Pull Request on GitHub

  1. Go to github.com/Diptendu123/git-training-demo
  2. Click "Compare & pull request" — GitHub shows this automatically after you push
  3. Set the title to: feat: add [your name] to team page
  4. Add a short description of your change
  5. Click "Create pull request"
7

Wait for review & merge

Your co-facilitator will review and merge your PR. Once merged, open the live page and watch your card appear. 🎉

Progress Checklist

1Cloned & on your branch

Run git branch — asterisk is on your branch, not main.

  • Seeing "not a git repository"? You need to cd git-training-demo first.
  • Still on main? Run git checkout -b add-your-name now.
2Card added and saved

Run git status — index.html should appear in red under "Changes not staged."

  • Nothing showing? The file wasn't saved — save it and check again.
  • Multiple files? Stage only index.html by name: git add index.html
3Committed and pushed

After commit, run git log --oneline — your commit appears at the top. After push, the branch appears on GitHub.

  • Push asking for credentials? Paste your PAT when prompted for a password.
  • "No upstream branch" error? Run git push origin add-your-name exactly.
4Pull request open

Your PR title is feat: add [your name] to team page and base branch is set to main.

  • Branch not visible on GitHub? Confirm the push completed without errors first.
  • Banner not appearing? Go to Pull requests tab → New pull request → select your branch.
⚡
Finished early? Level up.Leave a review comment on a teammate's PR — practice giving feedback. Try git log --oneline to see everyone's commits pile in. Or ask a colleague to review your PR before your co-facilitator does.
