The 4 Stages

✏️
Working Directory Where you write and edit files. Git watches but isn't saving anything yet.
📋
Staging Area Your selection queue. You decide exactly what goes into the next commit — nothing more.
💾
Local Repo The saved history on your machine. Safe, private, works without internet.
☁️
Remote Repo The shared copy on GitHub. This is how your work reaches your team.

The 3 Forward Commands

git add Working Directory → Staging Area "I want to include this change in my next commit."
git commit -m "" Staging Area → Local Repo "Save this snapshot with a label I can find later."
git push Local Repo → Remote Repo "Send my commits to GitHub so the team can see them."
⬇️
git pull Brings remote changes down to your local repo and working directory. Run this before starting new work.

5 Rules to Live By

1
Commit often, commit small One meaningful change per commit. Easier to review, easier to undo if something breaks.
2
Write commit messages a stranger can understand "Fix null reference in auth handler" beats "stuff" every time. Future-you is that stranger.
3
Always work on a branch, never directly on main A branch is a safe sandbox. Main should always contain working, reviewed code.
4
Pull before you push Sync the latest remote changes first. Prevents merge conflicts and saves everyone headaches.
5
Use git status constantly It tells you exactly where you are, what's staged, and what's not. When in doubt — run it.

What's Next

Put a real project under Git this week Even a folder of scripts counts. Run git init, make your first commit, and the workflow will start to feel natural fast.
Check the live team page If your PR was merged, open the repo and find the live page — your card should be there. If it's still open, leave a comment on a teammate's PR while you wait.
Explore branching strategies Feature branches, Gitflow, trunk-based development — the next level of collaboration. Ask your facilitator for a reading list.
Learn to resolve merge conflicts They sound scary, but once you've resolved one or two you'll see they're just a text edit with markers. It's a solvable problem, not a disaster.
The Point of All of This

You don't need to memorize every flag or option. You need to understand the mental model well enough to know what you're trying to do — then look up the exact syntax when you need it.
