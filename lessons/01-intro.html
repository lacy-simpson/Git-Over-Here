Intro

What to expect from this course.

What You'll Learn

  • 1Review what Git is and why it's useful
  • 2Set up Git and GitHub and verify your configuration
  • 3Run the basic Git workflow: status → add → commit → push
  • 4Create and use a branch so your work doesn't affect shared main
  • 5Open a pull request so a teammate can review your work before merge
  • 6Review best practices and mistakes to avoid

What This Course Is Not

This workshop is not intended to be a deep dive into the full breadth of Git. There are many excellent resources available if you want to build a stronger long-term foundation and go deeper into Git concepts, workflows, and edge cases.

This course is intentionally practical. The goal is to give you a quick overview of the basic functionality you need most, then help you use Git in a real application so the workflow feels useful and approachable right away.

Why This Course Matters

If you plan to work with Claude Code, Codex, or any other AI coding tool, Git is not optional background knowledge anymore. It is the skill that lets you safely review what the tool changed, save your progress, roll back mistakes, and share work with confidence instead of crossing your fingers.

For GPS, this becomes even more real. If you want to contribute to agents, skills, prompts, or other shared projects, Git is part of the job. It is how you show your work clearly, collaborate without stepping on someone else's changes, and participate in the same workflow the rest of the team will expect.

If you have ever thought, "I can probably get by without learning this part," this is the part that will eventually slow you down. The good news is that you do not need to master all of Git today. You just need enough of it to start working confidently in real situations, and that is exactly what this course is designed to help you do.

Pre-Reqs for Hands On

🖥️ New to the terminal? Open this first.
What is the terminal? A text-based window where you type commands instead of clicking buttons. Think of it as a faster way to navigate your computer.

How to open it:
Mac: Press Cmd+Space, type "Terminal," press Enter.
Windows: Press the Windows key, type "cmd" or "PowerShell," press Enter.

What you'll see: A blinking cursor waiting for you to type. That's normal.

Key commands to know:
pwd — shows your current folder location (Mac/Linux)
cd folder-name — moves into a folder
cd .. — moves up one folder level
Haven't installed Git yet? Open this.
Pick your operating system and follow the steps in order:
🍎 macOS
  1. Open Terminal — press Cmd+Space, type "Terminal," press Enter
  2. Check if Homebrew is installed: brew --version
    If you see "command not found," install Homebrew first: go to brew.sh, copy the install command on that page, paste it into your terminal, and press Enter. Wait for it to finish.
  3. Install Git: brew install git
  4. Verify: git --version — you should see a version number like git version 2.x.x
🪟 Windows
  1. Go to git-scm.com/download/win — the installer downloads automatically
  2. Run the installer and keep all default settings — click Next through each screen
  3. When the install finishes, open Git Bash (search for it in the Start menu) or open Command Prompt
  4. Verify: git --version — you should see a version number
🐧 Linux (Ubuntu / Debian)
  1. Open Terminal
  2. Run: sudo apt-get update && sudo apt-get install git — enter your password when prompted
  3. Verify: git --version — you should see a version number
Need to create your GitHub account? Open this.

Go to github.com and sign in or sign up. If you are signing up, use your uipath.com email if you don't already have an account.

  • Choose a professional username because you'll share it with teammates
  • Verify your email address before continuing
  • This creates a standard GitHub account, not an enterprise license
Important security note A standard GitHub account does not give you enterprise protections or approval to store sensitive information. Treat workshop and public-safe practice repos as non-sensitive spaces only: no client data, passwords, tokens, or real .env files.
Standard account vs. GitHub Enterprise
👤 Standard GitHub (you create today)
  • Free to create yourself
  • Good for this workshop and practice repos
  • Does not include enterprise access
🏢 GitHub Enterprise
  • Allocated separately by the org
  • Required for internal private repos
  • Not currently available to everyone
