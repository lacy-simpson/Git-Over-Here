What does "origin" mean?

origin is just a nickname — the default name Git gives to the remote repository you cloned from. It is not a special command or a fixed rule.

When you run git clone https://github.com/org/repo.git, Git automatically creates a remote named origin pointing to that URL. So when you later run git push origin main, you are saying: "push my commits to the remote called origin, onto the branch called main."

You can see all your remotes and their URLs by running git remote -v in your terminal.

What does cd do?

cd stands for "change directory." It moves your terminal into a folder — the same way double-clicking a folder in Finder or File Explorer opens it.

Common uses:

  • cd my-folder — move into a folder named "my-folder"
  • cd .. — move up one level (to the parent folder)
  • cd ~ — jump back to your home directory

To see where you currently are, run pwd (Mac/Linux) or type cd with no arguments (Windows).

Why does Git have a staging area?

Most save systems are all-or-nothing. Git's staging area gives you a "review queue" before you lock in a commit.

Real scenario: You edited three files today — two are part of one bug fix, and one is an unrelated experiment. Without staging, you'd have to commit all three together, making your history hard to read.

With staging, you git add the two related files, commit them as a clean, focused snapshot, then decide what to do with the third separately.

Think of it like putting items in a cart before checkout — you can review and remove before finalizing.

SSH Keys vs. Personal Access Tokens

Both methods authenticate you to GitHub — they're just different approaches with different tradeoffs.

Personal Access Token (PAT): A long string you paste as your "password" when Git asks. Easy to set up in 2 minutes. You may need to re-enter it depending on your credential settings.

SSH Key: A key pair generated on your machine. Once you add your public key to GitHub, Git never asks for credentials again. Takes ~5 minutes to set up but saves time long-term.

For this workshop: PAT is faster. For regular daily use: SSH is smoother.

git switch — the modern alternative

Modern Git (2.23+) introduced git switch as a cleaner, more focused alternative to git checkout for working with branches.

They are equivalent:

  • git checkout -b feature/name ≡ git switch -c feature/name
  • git checkout main ≡ git switch main

This guide uses git checkout because it works in all Git versions. If you see git switch in other tutorials, it's doing the same thing.

What is .gitignore?

A .gitignore file tells Git which files to never track — even if they exist in your project folder.

Common entries: .env files with secrets, node_modules/, compiled output, OS junk like .DS_Store.

Without it, git add . can accidentally stage things you never want in version control. It's one of the first files most developers add to a new project.

Working With a Local Repository

A local repository is the copy of your project that lives on your machine. This is where you initialize Git, inspect file changes, stage work, create commits, and create branches before anything is shared remotely.

That means a lot of Git happens without the internet. You can make commits, inspect history, and organize work locally first, then connect the repository to GitHub later when you are ready to push.

If the local workflow makes sense to you, the remote and collaboration workflow becomes much easier to understand.
